### Binance Public REST — klines (PRIMARY)
- **Host:** `api.binance.com`
- **Data:** Daily/intraday OHLCV candlesticks for BTC, ETH, SOL, BNB, XRP and all listed USDT/etc. spot pairs. Each kline includes openTime, OHLC, base volume, closeTime, quoteAssetVolume, numberOfTrades, takerBuy volumes.
- **Auth:** None (no API key). Keyless public market-data endpoint.
- **History:** BTCUSDT daily genesis verified = 2017-08-17 (startTime=0 -> ts 1502928000000, open 4261.48). Full continuous daily series since listing. Paginate via startTime/endTime, 1000 rows/call.
- **Coverage:** All major coins as USDT spot pairs (BTC/ETH/SOL/BNB/XRP verified). Hundreds of pairs. No native VND pair (BTCVND -> -1121 Invalid symbol).
- **Format:** JSON array of arrays. Index map: [0]openTime(ms), [1]open, [2]high, [3]low, [4]close, [5]volume(base asset), [6]closeTime(ms), [7]quoteAssetVolume(USDT), [8]numberOfTrades, [9]takerBuyBaseVolume, [10]takerBuyQuoteVolume, [11]ignore. Prices are strings; quote currency = trailing asset (USDT ~ USD).
- **Endpoints:** https://api.binance.com/api/v3/klines?symbol={SYMBOL}&interval={interval}&limit={1..1000}&startTime={ms}&endTime={ms}  | params: symbol (e.g. BTCUSDT, ETHUSDT, SOLUSDT, BNBUSDT, XRPUSDT), interval (1m,3m,5m,15m,30m,1h,2h,4h,6h,8h,12h,1d,3d,1w,1M — all verified live), limit (max 1000), startTime/endTime in epoch ms (startTime=0 returns genesis)
- **Terms:** robots.txt on api.binance.com is 'Disallow: /' (governs HTML crawlers, not this documented JSON data API). Public market-data endpoints are documented as keyless. Rate limit is request-weight based: response headers x-mb
```bash
curl -s 'https://api.binance.com/api/v3/klines?symbol=BTCUSDT&interval=1d&limit=3'
```
_proof:_ BTCUSDT 1d -> [[1781481600000,"65746.45","67292.15","65354.00","66328.74","18559.80102",1781567999999,"1230675240.53",3140494,...]] ; ETHUSDT 1d close "1777.29" ; genesis 2017-08-17 open 4261.48 ; intervals 1m/4h/1w/1M all returned a row; limit=1000 returned 1000 rows.

### Coinbase Exchange — product candles (SECONDARY, exchange-native USD)
- **Host:** `api.exchange.coinbase.com`
- **Data:** OHLCV candles for BTC-USD, ETH-USD and other USD/USDC products. True fiat USD pricing (not USDT).
- **Auth:** None for public market data (no key).
- **History:** Long (years) via start/end pagination, but only 300 candles per call (verified one call returned a ~350-row capped window: newest 1781654400 .. oldest 1751500800). Page backward with start/end to extend.
- **Coverage:** Major coins vs native USD (and USDC). No VND.
- **Format:** JSON array of arrays, newest-first: [ time(epoch SEC), low, high, open, close, volume(base) ]. NOTE order is low,high,open,close (not OHLC). Numbers are JSON numbers, USD.
- **Endpoints:** https://api.exchange.coinbase.com/products/{PRODUCT}/candles?granularity={sec}&start={ISO8601}&end={ISO8601}  | granularity in seconds: 60,300,900,3600,21600,86400 (86400=daily). product e.g. BTC-USD, ETH-USD, SOL-USD. Max 300 candles per request.
- **Terms:** api.exchange.coinbase.com/robots.txt returns JSON {"message":"Unauthorized."} (no crawl policy served). Public market-data is documented & keyless; ~10 req/s public rate limit per Coinbase docs. Check Terms before redist
```bash
curl -s 'https://api.exchange.coinbase.com/products/BTC-USD/candles?granularity=86400' -H 'User-Agent: Mozilla/5.0'
```
_proof:_ BTC-USD daily -> [[1781654400,64491.88,66384.01,65616.64,65794.87,4159.143],[1781568000,65300.03,66944,66276.79,65616.64,6347.81]] (order = ts,low,high,open,close,vol)

### Kraken — public OHLC (TERTIARY, exchange-native USD)
- **Host:** `api.kraken.com`
- **Data:** OHLC candles for XBT/USD (BTC), ETH/USD etc. Includes VWAP and trade count.
- **Auth:** None (public endpoint).
- **History:** Daily call returned ~721 rows (oldest 1719446400 = 2024-06, newest 1781654400). 'since' shifts the window but Kraken caps returned count (~720 candles). Shorter usable daily history than Binance/Coinbase.
- **Coverage:** Major coins vs native USD/EUR. No VND.
- **Format:** JSON {error:[],result:{<PAIR>:[[ time(epoch SEC), open, high, low, close, vwap, volume, count ], ...], last:<id>}}. Prices are strings, USD.
- **Endpoints:** https://api.kraken.com/0/public/OHLC?pair={PAIR}&interval={min}&since={epoch}  | interval in MINUTES: 1,5,15,30,60,240,1440(daily),10080,21600. pair e.g. XBTUSD (BTC), ETHUSD. Result key is canonical name (XXBTZUSD / XETHZUSD).
- **Terms:** api.kraken.com has no robots blocker (returns API error for /robots.txt). Public market data documented & keyless; rate-limited per Kraken API tiers. Check Terms before redistribution.
```bash
curl -s 'https://api.kraken.com/0/public/OHLC?pair=XBTUSD&interval=1440'
```
_proof:_ XXBTZUSD daily -> [1719446400,"60815.2","62297.8","60580.7","61617.1","61491.3","878.31549593",20197] ; XETHZUSD -> [1719446400,"3367.67","3470.72","3362.00","3445.39",...]

### Yahoo Finance — chart API (FALLBACK, longest history + fiat breadth)
- **Host:** `query1.finance.yahoo.com`
- **Data:** Daily OHLCV for BTC-USD, ETH-USD and crypto pairs incl. some fiat (e.g. BTC-EUR); also FX like USDVND=X to derive VND prices. Rich meta block (firstTradeDate, 52wk high/low, regularMarketPrice).
- **Auth:** None, but UNOFFICIAL endpoint and aggressively rate-limited (intermittent HTTP 429). Use desktop User-Agent + retry with jitter; query2 host as alternate.
- **History:** Deepest of all: BTC-USD firstTradeDate=1410912000 = 2014-09-17. range=max gives full history.
- **Coverage:** Broadest fiat coverage (USD/EUR/etc.) + FX pairs (USDVND=X) enabling VND derivation. Major coins covered.
- **Format:** JSON chart.result[0]: meta{currency,symbol,firstTradeDate,regularMarketPrice,fiftyTwoWeekHigh/Low...}, timestamp[](epoch SEC), indicators.quote[0]{open[],high[],low[],close[],volume[]}, indicators.adjclose[0].adjclose[]. Units USD (per symbol currency).
- **Endpoints:** https://query1.finance.yahoo.com/v8/finance/chart/{SYMBOL}?interval={1d|1wk|1mo}&range={1mo|1y|5y|max}  (or period1/period2 epoch SEC). SYMBOL e.g. BTC-USD, ETH-USD, SOL-USD, BTC-EUR, USDVND=X. Alternate host: query2.finance.yahoo.com.
- **Terms:** query1.finance.yahoo.com/robots.txt = 'Disallow: /'. Endpoint is unofficial/undocumented and rate-limited (429 seen repeatedly). Acceptable as cross-check/fallback; NOT recommended as sole production source. Verify Yahoo
```bash
curl -s 'https://query1.finance.yahoo.com/v8/finance/chart/BTC-USD?interval=1d&range=1mo' -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/124.0 Safari/537.36'
```
_proof:_ BTC-USD 1d rows: 1781481600 O65711.11 H67248.13 L65315.84 C66289.50 V32927321950 ; 1781654400 O65605.45 H66231.34 L64545.50 C65926.22 ; meta firstTradeDate 1410912000, fiftyTwoWeekHigh 126198.07.

## Notes

All endpoints were learned from general knowledge of public exchange/data APIs and hit on each provider's OWN server directly, then empirically verified with live curl. VND PAIRS: No major exchange offers native crypto/VND klines. Binance spot has no BTCVND/ETHVND symbol (verified: returns -1121 "Invalid symbol"). Binance P2P (USDT/VND) exists but is order-book/advertisement data, not OHLC klines, and is not a clean EOD time series. RECOMMENDED PATH for VND: take USD price from any verified source above and multiply by a USD/VND FX series (Yahoo USDVND=X works when not rate-limited; or an official SBV/bank FX series). So VND is derivable, not natively available as crypto-pair candles.
