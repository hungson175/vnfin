### VPS histdatafeed (TradingView-UDF) — index value history
- **Host:** `histdatafeed.vps.com.vn`
- **Data:** Daily/intraday OHLCV index VALUE series. Widest coverage: VNINDEX, VN30, HNXINDEX, HNX30, UPCOMINDEX, VNALL(=VNALLSHARE), VNXALL(=VNXALLSHARE), VN100, VNDIAMOND, VNFINLEAD, VNMID, VNSML, and all 10 sector indices (VNCOND,VNCONS,VNENE,VNFIN,VNHEAL,VNIND,VNIT,VNMAT,VNREAL,VNUTI).
- **Auth:** None (no-auth, public). Requires -4 (IPv6 hangs) + browser User-Agent.
- **History:** Deepest: 6325 daily bars, first=2000-07-28 (VNINDEX base=100), last=current (2026-06-17 c=1806.2).
- **Coverage:** HOSE, HNX, UPCOM main indices + VN-sector + thematic indices. resolution=D verified; intraday resolutions advertised as supported.
- **Format:** TradingView-UDF JSON: {symbol, s:'ok'|'no_data', t:[epoch_sec], o:[],h:[],l:[],c:[], v:[]}. Index values in points; v in shares.
- **Endpoints:** GET https://histdatafeed.vps.com.vn/tradingview/history?symbol={SYMBOL}&resolution={1|5|15|30|60|D|W|M}&from={epoch_sec}&to={epoch_sec}  ;  symbol metadata: GET /tradingview/symbols?symbol={SYMBOL}
- **Terms:** No robots.txt (404 on /robots.txt — pure JSON API host, no crawl policy). Public broker chart feed; rate-limit politely. No explicit redistribution license — treat as read-only reference, attribute/derive cautiously.
```bash
curl -4 -s -m 25 -A 'Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 Chrome/124.0 Safari/537.36' "https://histdatafeed.vps.com.vn/tradingview/history?symbol=VNINDEX&resolution=D&from=$(date -d '2024-06-10' +%s)&to=$(date -d '2024-06-14' +%s)"
```
_proof:_ {"symbol":"VNINDEX","s":"ok","t":[1717977600,...],"c":[1290.67,1284.41,1300.19,1301.51,1279.91],"o":[1292.67,...]}  (identical to SSI & VND for same dates)

### SSI iBoard charts (TradingView-UDF) — index value history
- **Host:** `iboard-api.ssi.com.vn`
- **Data:** Daily OHLCV index VALUE series. Verified symbols: VNINDEX, VN30, HNXINDEX (case-sensitive!), HNX30, VNALL(=VNALLSHARE), VNXALL, VNIT + other sectors. NOTE: UPCOM via SSI (HNXUpcomIndex) returns WRONG 0.1 values — use VPS/VND for UPCOM.
- **Auth:** None (no-auth, public). Requires -4 + browser UA.
- **History:** Deep: 4795 daily bars, first=2000-07-28 (c=100), last=2026-06-17 (c=1806.2).
- **Coverage:** HOSE/HNX main indices + VN sector indices. Symbol casing matters (HNXINDEX correct; HNXIndex returns a different junk series).
- **Format:** {code:'SUCCESS', message, data:{t:[],o:[],h:[],l:[],c:[],v:[], s:'ok', nextTime}}. Index values in points.
- **Endpoints:** GET https://iboard-api.ssi.com.vn/statistics/charts/history?resolution={1D|1|...}&symbol={SYMBOL}&from={epoch_sec}&to={epoch_sec}
- **Terms:** No robots.txt (404). Public SSI iBoard backend. No stated redistribution terms — read-only reference use.
```bash
curl -4 -s -m 25 -A 'Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 Chrome/124.0 Safari/537.36' "https://iboard-api.ssi.com.vn/statistics/charts/history?resolution=1D&symbol=VNINDEX&from=$(date -d '2024-06-10' +%s)&to=$(date -d '2024-06-14' +%s)"
```

### SSI iBoard-query stock group — index CONSTITUENTS (members)
- **Host:** `iboard-query.ssi.com.vn`
- **Data:** Full member-symbol list per index group (constituents, NOT weights). Verified: VN30(30), HNX30(30), VN100(100), VNMID(70), VNSML(200), VNDIAMOND(18), VNFINLEAD(23), VNFINSELECT(41); plus full exchange membership VNINDEX(673 HOSE), HNXIndex(300).
- **Auth:** None (no-auth, public). Requires -4 + browser UA.
- **History:** Snapshot of CURRENT membership only (no historical basket).
- **Coverage:** HOSE+HNX index groups and thematic/sector baskets. Gives membership; does NOT include per-stock index weight.
- **Format:** {code:'SUCCESS', message, data:[ {stockSymbol, exchange, isin, companyNameEn, companyNameVi, refPrice, ceiling, floor, matchedPrice, ...full quote fields} ]}. One object per constituent.
- **Endpoints:** GET https://iboard-query.ssi.com.vn/stock/group/{GROUP}  where GROUP in {VN30,HNX30,VN100,VNMID,VNSML,VNDIAMOND,VNFINLEAD,VNFINSELECT,VNINDEX,HNXIndex,...}
- **Terms:** No robots.txt (404). Public SSI iBoard query backend. WEIGHTS not exposed here — see notes: no clean JSON weights source found (HOSE/VietStock/CafeF/finfo all dead-ended); official weights only in HOSE periodic PDF/Excel
```bash
curl -4 -s -m 25 -A 'Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 Chrome/124.0 Safari/537.36' "https://iboard-query.ssi.com.vn/stock/group/VN30"
```
