### Fmarket - Fund List (filter)
- **Host:** `api.fmarket.vn`
- **Data:** FUND LIST: all VN open-ended mutual funds distributed on Fmarket (equity/bond/balanced) with code, shortName, internal id, current nav (VND/unit), lastYearNav, managementFee, avgAnnualReturn, owner (fund manager), firstIssueAt. Supports filtering by fundAssetTypes (e.g. STOCK) and free-text searchField.
- **Auth:** None (no token/cookie required)
- **History:** Current snapshot (list of funds); per-fund history via NAV endpoint below.
- **Coverage:** 65 funds total. Verified codes include VEOF, VESAF, VFF, VIBF, SSISCA, BVPF, VLBF, VFMVF1, VNDAF, VCBF*, MB*, MAF*, etc. STOCK filter -> 32 equity funds.
- **Format:** JSON: {status:200, data:{total:65, page, pageSize, rows:[{id,code,shortName,name,nav,lastYearNav,managementFee,avgAnnualReturn,owner:{name,shortName},...}]}}. nav in VND per fund unit.
- **Endpoints:** POST https://api.fmarket.vn/res/products/filter  body(JSON): {"types":["NEW_FUND","TRADING_FUND"],"sortField":"navTo6Months","sortOrder":"DESC","page":1,"pageSize":100,"isIpo":false,"fundAssetTypes":[],"searchField":""}  (fundAssetTypes can be ["STOCK"]; searchField does name/code search)
- **Terms:** fmarket.vn/robots.txt disallows only /assets/params/ and /blog|help-center/ket-qua-tim-kiem/* (search pages); api.fmarket.vn has no robots.txt. Public read endpoint powering the fund-browse UI. Commercial distribution pl
```bash
curl -4 -s -m 25 -X POST 'https://api.fmarket.vn/res/products/filter' -H 'User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 Chrome/124.0 Safari/537.36' -H 'Content-Type: application/json' --data '{"types":["NEW_FUND","TRADING_FUND"],"sortField":"navTo6Months","page":1,"pageSize":100,"isIpo":false,"fundAssetTypes":[],"searchField":""}'
```
_proof:_ status 200, total 65; rows e.g. id=38 VNDAF nav=19626.37; id=20 VEOF nav=34942.66; id=23 VESAF nav=33779.47; id=11 SSISCA nav=43503.81. STOCK filter total=32. searchField VESAF -> id=23 'QUY DAU TU CO PHIEU TANG TRUONG CHIEN LUOC'.

### Fmarket - NAV History
- **Host:** `api.fmarket.vn`
- **Data:** NAV HISTORY time series for a single fund (daily/business-day frequency). One row per available NAV date.
- **Auth:** None
- **History:** Multi-year, from fund inception. Verified: VEOF=1729 rows; VESAF=1317 rows (from 2017-04-25 at 10000 VND); VNDAF=1267 rows (from 2018-01-12). Use isAllData:1 for full history, or fromDate/toDate (YYYY-MM-DD) to window it
- **Coverage:** All 65 Fmarket funds (use the internal id from the fund-list endpoint as productId).
- **Format:** JSON: {status:200, data:[{id, navDate:'YYYY-MM-DD', nav: <float VND/unit>, productId, createdAt:<epoch ms or null>}]}. nav = NAV per fund unit in VND.
- **Endpoints:** POST https://api.fmarket.vn/res/product/get-nav-history  body(JSON): {"isAllData":1,"productId":<id>,"fromDate":"2018-01-01","toDate":"2026-06-18"}  (omit isAllData and set fromDate/toDate to fetch a date range)
- **Terms:** Same host/ToS note as fund-list. No auth. navDate is string YYYY-MM-DD; createdAt epoch millis (sometimes null for old rows).
```bash
curl -4 -s -m 25 -X POST 'https://api.fmarket.vn/res/product/get-nav-history' -H 'User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 Chrome/124.0 Safari/537.36' -H 'Content-Type: application/json' --data '{"isAllData":1,"productId":20,"fromDate":"2018-01-01","toDate":"2026-06-18"}'
```
_proof:_ productId=20 (VEOF) -> rows 1729, last {navDate:'2025-12-05', nav:36153.51, productId:20}. productId=23 (VESAF) -> rows 1317, first {navDate:'2017-04-25', nav:10000.0}. productId=38 (VNDAF) -> rows 1267 from 2018-01-12.

### Fmarket - Fund Detail (Holdings / Allocation)
- **Host:** `api.fmarket.vn`
- **Data:** HOLDINGS & ALLOCATION for a single fund: top holdings (per-stock weight + industry + last price), asset-class allocation (STOCK/CASH/BOND %), and industry/sector allocation %. Plus full fund metadata (fees, inception, description, manager).
- **Auth:** None
- **History:** Latest disclosed snapshot (holdings updated periodically, updateAt epoch ms on each row). Not a time series.
- **Coverage:** All 65 Fmarket funds via GET /res/products/{id}. Equity holdings in productTopHoldingList; per-bond holdings in productTopHoldingBondList (bond funds populate only this one); asset-class split in productAssetHoldingList. So bond funds DO disclose per-security bond holdings (live-probe, #173) — not only a bond/cash allocation %.
- **Format:** JSON: data.productTopHoldingList:[{stockCode, netAssetPercent, industry, price, type:'STOCK', updateAt}], data.productTopHoldingBondList:[{stockCode(bond code e.g. BAF126003), netAssetPercent, industry, price:null, type:'BOND', updateAt}], data.productAssetHoldingList:[{assetType:{code:'STOCK'|'CASH'|'BOND',name}, assetPercent, updateAt}], data.productIndustriesHoldingList:[{industry, assetPercent}]. Percentages 0-100. (No productBondHoldingList key — the bond array is productTopHoldingBondList.)
- **Endpoints:** GET https://api.fmarket.vn/res/products/{id}   (id = internal fund id from the fund-list endpoint, e.g. 20=VEOF, 38=VNDAF)
- **Terms:** Same host/ToS note as above. No auth. Holdings reflect last periodic fund disclosure.
```bash
curl -4 -s -m 25 'https://api.fmarket.vn/res/products/20' -H 'User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 Chrome/124.0 Safari/537.36' -H 'Accept: application/json'
```
_proof:_ id=20 (VEOF): top holding {stockCode:'MBB', netAssetPercent:7.99, industry:'Ngan hang'}; asset alloc STOCK 97.44%, CASH 2.56%. id=38 (VNDAF): top {EIB 13.53%, VCG 13.2%, NVL 11.82%}; asset STOCK 78.85% / CASH 21.15%; industries Banking 31.48%, RealEstate 20.59%, Construction 17.71%.
