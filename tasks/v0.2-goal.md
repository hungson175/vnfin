## Phases & acceptance criteria

### Phase A — Stability (first; all offline-testable)
- **A1 Public-API snapshot** — `scripts/dump_api_surface.py`, `tests/test_public_api_surface.py`,
  `tests/snapshots/public_api_v0_2_0.json`. Tier-0 surface: top-level + per-domain `__all__`,
  factory/convenience signatures (`client/source/history/index_history/get_financials/get_indicator/...`),
  public frozen-dataclass fields (name/order/normalized-type/default-presence/frozen),
  public enum members+values (`Interval, AdjustmentPolicy, MacroIndicator, Frequency, StatementType, Period`),
  public client/source user-facing methods, canonical unit/currency defaults.
  **Compatibility-aware comparator** — hard-fail on remove/rename/reorder/kind-change/required-param-add/unit-change;
  allow+report additive changes. **Done:** test green; intentional changes update the snapshot consciously.
- **A2 `docs/stability.md`** — SemVer (patch=no-break, minor=additive, major=break),
  `DeprecationWarning` + documented replacement + ≥1 minor-release window.
- **A3 Health core** — private `vnfin/_health.py` (`SourceHealth` w/ status enum
  ok|degraded|down|host_blocked|skipped, reachable/schema_ok/value_sane/latency_ms/checked_at_utc/error_type/note/metadata-sanitized)
  + thin `scripts/healthcheck.py` CLI. Offline unit tests (mocked HTTP). Small critical probe set
  (prices SSI, fundamentals VNDirect, gold BTMC, crypto Binance, macro World Bank, FX primary —
  not every source polished). Schema-drift = required-paths/types + semantic invariants; cross-source
  parity only where independent same-unit sources exist; single-source domains use schema + value bands.
  `STATUS.md` generated from sanitized timestamped JSON; **not auto-pushed**. Cron = DESIGN ONLY (hold for Boss).

### Phase B — FX MVP (narrow, daily/current)
- **B0 source-vetting+design doc** `docs/design/fx-sources.md` → **reviewer design gate (APPROVE)** before any adapter code.
  Sources: `open.er-api.com` (primary, no-key, USD-base incl. VND ⇒ cross-rates) + Vietcombank XML
  (failover, no-key, official). Both quote **VND per 1 unit foreign currency** → unit guard trivial.
  Daily/current only (no history, no broad pair universe, no total-return). SBV = candidate only (no machine API).
- **B1 implement** `vnfin/fx/` — typed `FXRate` (base, quote, rate, unit="VND per 1 USD"-style, as_of, source),
  ≥2 adapters, FailoverClient reuse + unit-homogeneity guard, synthetic fixtures, `docs/sources/fx-*.md`.
  Opt-in live USD/VND cross-source parity test in `live_tests/`. **reviewer impl gate** after green.

### Phase C — Dividends (research/design ONLY)
- `docs/design/corporate-actions.md` — design a clean-room adapter against VNDirect finfo `/v4/events`
  (DIVIDEND/STOCKDIV/KINDDIV; effectiveDate=ex, expiredDate=record, actualDate=pay; dividend=VND/share, ratio=%par).
  Single-source (accepted, like funds). Runtime-fetch-only, MEDIUM redistribution flagged. **No adapter code merged.**

### Phase D — Close-out
- Update README/api.md/roadmap.md + new `CHANGELOG.md`; bump pyproject `version="0.2.0"`.
- Full green offline (`--cov-fail-under=85`) + `python -m build`. **reviewer final release-readiness gate.**
- **HOLD for Boss:** version tag, push, PyPI, crontab enablement. End-state = "release-ready, not released."
